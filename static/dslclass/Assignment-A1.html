Assignment A1 - Fluent APIs

In this assignment, you'll create a simple fluent API to create a limited subset of Java Swing GUIs.

Grab the starter code from Assignment1.kt

Create an implementation of the missing GuiBuilder class. Do not use Progressive Interfaces or Child Builders for your implementation.

Note that your implementation will need to do some error checking to validate the order of function calls. This will be runtime validation; you can throw exceptions if the GUI is in error. Some cases to consider:

  • north(), south(), east(), west() and center() can only be called if the current layout is a BorderLayout. (The top-level is automatically a BorderLayout)
  • After calling north(), south(), east(), west() and center(), a component (button(), grid(), flow() or border()) must be the next function called
  • grid(), flow() and border() must be ended with an end() call
  • No builder functions can be called after show()

After implementing the GuiBuilder, test it with 3-4 GUIs that you define. These GUIs should be submitted with your code.

After testing, answer the following questions and submit with your code:

  • Compared to the "raw" approach at the top of the app

    • How do you think your implementation affects performance?
    • How do you think your implementation affects maintainability (consider the builder and use of the builder separately)?

  • Describe a case where the GUI would be ambiguous if we did not have the end() call

  • Could this builder language be implemented using Progressive Interfaces or Child Builders? If no, why not?
    • As-is?
    • With minor changes?

  • If we added new components (like a TextField), how would the following be impacted? (For this question, assume all could be used to implement this language)

    • Your implementation
    • A Progressive Interface implementation
    • A Child Builder implementation

  • From the point of view of the user of your builder, how would a Progressive Interface or Child Builder make his work easier or more difficult?

Submitting Your Assignment

To submit your assignment (do this outside IntelliJ/Android Studio):

  1. Close the project in IntelliJ/Android Studio (and/or quit IntelliJ/Android Studio)
  2. Find the directory for your project
  3. Delete the build and app/build directories from the project folder
  4. Zip the project directory. Be sure to get all files and subdirectories, including those that start with '.'. Be sure to name the zip file lastname.firstname.hw1
  5. Go to the Blackboard course website
  6. Click on the Assignment Submission link on the left navigation bar
  7. Click on the Assignment A1 link
  8. Attach your submission and enter any notes
  9. Be sure to submit your assignment, not just save it!
